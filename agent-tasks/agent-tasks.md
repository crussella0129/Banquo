# Agent Tasks (Persistent Backlog)

- [ ] (candidate, future sprint): Job-object breakaway for full launch-independence. The release binary is already independent of a *plain* shell (verified: survives the launching PowerShell exiting). BUT when launched via PATH **from inside Windows Terminal**, WT parents children in a job object with `JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE`, so closing that WT tab can still kill Banquo. Robust fix = break away from the parent job on startup (`CREATE_BREAKAWAY_FROM_JOB` / `AssignProcessToJobObject`), which needs Win32 calls → `unsafe` (ADR-002 `#![forbid(unsafe_code)]` tension, same family as the default-terminal handoff). Workaround today: launch from the Start-menu shortcut / double-click (parent = explorer, no job) — fully independent. Decide unsafe-carve-out vs. helper-process when scheduled.
